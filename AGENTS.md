# 프로젝트 규칙 (AI·에이전트용)

이 문서는 이 저장소에서 코드를 작성하거나 수정할 때 **반드시 따를 규칙**을 정리한 것이다. 사용자가 매번 스택과 방식을 설명하지 않아도 되도록, 아래를 기본 전제로 한다.

## 기술 스택

| 영역 | 선택 |
|------|------|
| 프레임워크 | **Next.js** — **App Router** (`app/` 디렉터리)만 사용 |
| 언어 | **TypeScript** (`.ts`, `.tsx`) |
| 스타일 | **Tailwind CSS** |
| 데이터베이스·백엔드(BaaS) | **Supabase** (클라이언트·서버 클라이언트 구분 등 Supabase 관례 준수) |

---

## 반드시 지켜야 할 것 (DO)

- **App Router**로 라우팅·레이아웃·로딩·에러 UI를 구성한다. `pages/` 기반 Pages Router는 새 코드에 사용하지 않는다.
- 모든 새 컴포넌트와 유틸은 **TypeScript**로 작성하고, `any` 남발 없이 타입을 명시하거나 추론되게 작성한다.
- 스타일은 **Tailwind 유틸리티 클래스**를 우선 사용한다. 프로젝트에 이미 있다면 `tailwind.config`, 전역 스타일과의 일관성을 맞춘다.
- 데이터 접근은 **Supabase** 클라이언트(브라우저용·서버용 구분)를 사용하고, 환경 변수(`NEXT_PUBLIC_*`, 서버 전용 키)는 Next.js 규칙에 맞게 둔다.
- 서버에서만 실행해야 하는 비밀 처리·관리자 작업은 **Route Handler**(`app/api/.../route.ts`) 또는 **Server Actions** 등 서버 경로에 둔다.
- **React Server Components**를 기본으로 두고, 클라이언트 전용이 필요할 때만 `"use client"`를 파일 상단에 붙인다.
- 접근성·시맨틱 HTML·반응형 레이아웃을 기본으로 고려한다.

---

## 하면 안 되는 것 (DON'T)

- **Pages Router**(`pages/` 디렉터리)로 새 페이지나 API를 추가하지 않는다.
- **JavaScript만 있는 파일**으로 새 기능을 추가하지 않는다. (기존 JS 레거시가 있으면 점진적 마이그레이션은 별도 합의.)
- **인라인 `style={{}}`**로 레이아웃·테마를 반복 구현하지 않는다. (Tailwind로 불가능한 예외만 최소한으로.)
- **Supabase 서비스 롤 키**를 클라이언트 번들에 노출하거나 `NEXT_PUBLIC_`에 두지 않는다.
- **환경 변수·토큰**을 코드에 하드코딩하지 않는다.
- 불필요한 **`"use client"`**로 전체 트리를 클라이언트 컴포넌트로 만들지 않는다.
- 사용자 요청 범위를 넘는 **대규모 리팩터·무관한 파일 수정**을 하지 않는다.

---

## 참고

- Next.js·Supabase 버전은 `package.json`을 기준으로 한다. 모호하면 App Router + TypeScript + Tailwind + Supabase 조합에 맞는 최신 공식 문서 패턴을 따른다.
